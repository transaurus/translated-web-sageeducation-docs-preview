🚀 **コントリビューターの皆さん、ようこそ！** 🚀

Sage WebUIへの貢献に興味を持っていただき、誠にありがとうございます。このドキュメントは、あなたの貢献がプロジェクトを効果的に強化できるよう、プロセスを案内するためのものです。一緒にSage WebUIをさらに良いものにしていきましょう！

## 💡 貢献について

貢献をお考えですか？素晴らしい！以下が貢献方法です：

### 🌟 コード貢献ガイドライン

プルリクエストは歓迎します。提出前に以下の点をご確認ください：

1. アイデアについてのディスカッションを[こちら](https://github.com/Startr/AI-WEB-openwebui/discussions/new/choose)で開いてください。
2. プロジェクトのコーディング標準に従い、新機能にはテストを含めてください。
3. 必要に応じてドキュメントを更新してください。
4. 明確で説明的なコミットメッセージを書いてください。

### 🛠 コードPRのベストプラクティス：

1. **原子性のあるPR:** PRは小さく、焦点を絞り、単一の目的またはタスクに対応するようにしてください。これによりコードレビューが容易になり、無関係な問題が混入する可能性を抑えられます。変更の範囲が大きくなりすぎた場合は、論理的に関連のない小さなPRに分割することを検討してください。
2. **既存のコード規約に従う:** コードがプロジェクトの既存のコーディング標準と慣行に沿っていることを確認してください。
3. **外部依存関係の追加を避ける:** 事前の議論なしに追加の外部依存関係を含めないでください。
4. **フレームワークに依存しないアプローチ:** 私たちはフレームワークに依存しないことを目指しています。可能な限り外部フレームワークやライブラリに頼るのではなく、独自に機能を実装してください。このアプローチについて疑問や提案がある場合は、遠慮なく議論してください。

貢献いただきありがとうございます！🚀

### 📚 ドキュメント & チュートリアル

ドキュメントの改善、チュートリアルの執筆、またはWeb UIのセットアップと最適化に関するガイドの作成を通じて、Sage WebUIをよりアクセスしやすいものにするお手伝いをしてください。

### 🌐 翻訳と国際化

Sage WebUIをより広い対象者に利用可能にするお手伝いをしてください。このセクションでは、プロジェクトに新しい翻訳を追加するプロセスを案内します。

翻訳はJSONファイルで保存しています。既存の翻訳ファイルは`src/lib/i18n/locales`ディレクトリにあります。各ディレクトリは特定の言語に対応しており、例えば`en-US`は英語（米国）、`fr-FR`はフランス語（フランス）などです。特定の言語に適したコードを見つけるには、[ISO 639言語コード](http://www.lingoes.net/en/translator/langcode.htm)を参照してください。

新しい言語を追加するには：

- `src/lib/i18n/locales`パスに、適切な言語コードを名前とした新しいディレクトリを作成してください。例えば、スペイン語（スペイン）の翻訳を追加する場合、`es-ES`という名前の新しいディレクトリを作成します。
- 米国英語の翻訳ファイル（`src/lib/i18n/locale`内の`en-US`ディレクトリから）をこの新しいディレクトリにコピーし、JSON形式で文字列値をあなたの言語に合わせて更新してください。JSONオブジェクトの構造は保持するようにしてください。
- 言語コードとそのタイトルを`src/lib/i18n/locales/languages.json`の言語ファイルに追加してください。

### 🤔 質問 & フィードバック

質問やフィードバックがありますか？[Discordコミュニティ](https://discord.com/channels/1074411230040703046/1074411231030550632)に参加するか、イシューを開いてください。お手伝いします！

### 🚨 問題の報告

何かおかしな点に気付きましたか？アイデアがありますか？[Issuesタブ](https://github.com/Startr/AI-WEB-openwebui/issues)をチェックして、既に報告または提案されていないか確認してください。なければ、新しいイシューを開いてください。問題を報告する際は、私たちのイシューテンプレートに従ってください。これらのテンプレートは、最初から必要な詳細がすべて提供されるように設計されており、あなたの懸念をより効率的に対処できるようにします。

:::important

- **テンプレートの遵守:** 提供されたIssueテンプレートに従わない場合、または要求された情報を一切提供しない場合、Issueは追加の検討なしにクローズされる可能性があります。このアプローチは、Issue追跡の管理性と整合性を維持するために重要です。

- **詳細が鍵:** Issueが理解され、効果的に対処されるためには、包括的な詳細を含めることが不可欠です。説明は明確にし、再現手順、期待される結果、実際の結果を含めてください。十分な詳細がないと、Issueの解決が妨げられる可能性があります。

:::

### 🧭 サポート範囲

Sage WebUI自体ではなく、特にDockerセットアップなど実行環境に関連するIssueが増加していることに気付いています。Dockerデプロイメントをサポートするよう努めていますが、スムーズな体験のためにはDockerの基礎知識が重要です。

- **Dockerデプロイメントのサポート:** Sage WebUIはDockerデプロイメントをサポートしています。Dockerの基礎知識があることを前提としています。Dockerの基本については、[公式Dockerドキュメント](https://docs.docker.com/get-started/overview/)を参照してください。

- **高度な設定:** HTTPS用のリバースプロキシの設定やDockerデプロイメントの管理には基礎知識が必要です。これらのスキルを学ぶためのオンラインリソースは多数あります。この知識を身につけることで、Sage WebUIや類似プロジェクトでの体験が大幅に向上します。
