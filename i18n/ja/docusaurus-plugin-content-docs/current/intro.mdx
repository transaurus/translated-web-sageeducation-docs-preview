# Sage WebUI

## Dockerでクイックスタート 🐳

:::info

Sage Open WebUIが正しく動作するには、**WebSocket**のサポートが必要です。ネットワーク設定でWebSocket接続が許可されていることを確認してください。

:::

**Ollamaがローカルコンピュータ上にある場合**、次のコマンドを使用します：

```bash
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v sage-open-webui:/app/backend/data --name sage-open-webui --restart always ghcr.io/Startr/AI-WEB-openwebui:main
```

**Nvidia GPUサポート付きでSage WebUIを実行するには**、次のコマンドを使用します：

```bash
docker run -d -p 3000:8080 --gpus all --add-host=host.docker.internal:host-gateway -v sage-open-webui:/app/backend/data --name sage-open-webui --restart always ghcr.io/Startr/AI-WEB-openwebui:cuda
```

### OllamaとバンドルされたSage WebUI

このインストール方法では、Sage WebUIとOllamaを単一のコンテナイメージにバンドルしており、1つのコマンドでシームレスにセットアップできます。ハードウェア構成に応じて適切なコマンドを選択してください：

- **GPUサポートあり**:
  GPUリソースを活用するには、次のコマンドを実行します：

  ```bash
  docker run -d -p 3000:8080 --gpus=all -v ollama:/root/.ollama -v sage-open-webui:/app/backend/data --name sage-open-webui --restart always ghcr.io/Startr/AI-WEB-openwebui:ollama
  ```

- **CPUのみ**:
  GPUを使用しない場合は、代わりにこのコマンドを使用します：

  ```bash
  docker run -d -p 3000:8080 -v ollama:/root/.ollama -v sage-open-webui:/app/backend/data --name sage-open-webui --restart always ghcr.io/Startr/AI-WEB-openwebui:ollama
  ```

どちらのコマンドも、Sage WebUIとOllamaの組み込みインストールを容易にし、迅速なセットアップを可能にします。

インストール後、Sage WebUIには[http://localhost:3000](http://localhost:3000)でアクセスできます。お楽しみください！😄

### Devブランチを使用する 🌙

:::warning
`:dev`ブランチには最新の不安定な機能や変更が含まれています。バグや未完成の機能がある可能性があるため、自己責任で使用してください。
:::

最新の最先端機能を試したい場合で、時折の不安定さを許容できる場合は、以下のように`:dev`タグを使用できます：

```bash
docker run -d -p 3000:8080 -v sage-open-webui:/app/backend/data --name sage-open-webui --restart always ghcr.io/Startr/AI-WEB-openwebui:dev
```

### Sage WebUIの更新

Sage WebUIコンテナを簡単に更新するには、以下の手順に従ってください：

#### 手動更新

[Watchtower](https://containrrr.dev/watchtower)を使用してDockerコンテナを手動で更新します：

```bash
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock containrrr/watchtower --run-once sage-open-webui
```

#### 自動更新

コンテナを5分ごとに自動更新するには：

```bash
docker run -d --name watchtower --restart unless-stopped -v /var/run/docker.sock:/var/run/docker.sock containrrr/watchtower --interval 300 sage-open-webui
```

🔧 **注意**: コンテナ名が異なる場合は、`sage-open-webui`を実際のコンテナ名に置き換えてください。

## 手動インストール

Sage WebUIをインストールして実行する主な方法は2つあります：`uv`ランタイムマネージャーを使用する方法と、Pythonの`pip`を使用する方法です。どちらの方法も有効ですが、**環境管理を簡素化し競合を最小限に抑えるため、`uv`の使用を強く推奨します**。

### `uv`を使用したインストール（推奨）

`uv`ランタイムマネージャーは、Sage WebUIのようなアプリケーションのためのシームレスなPython環境管理を保証します。開始するには以下の手順に従ってください：

#### 1. `uv`のインストール

お使いのオペレーティングシステムに適したインストールコマンドを選択してください：

- **macOS/Linux**:  
  ```bash
  curl -LsSf https://astral.sh/uv/install.sh | sh
  ```

- **Windows**:  
  ```powershell
  powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
  ```

#### 2. Sage WebUIの実行

`uv`がインストールされたら、Sage WebUIの実行は簡単です。以下のコマンドを使用し、データ損失を防ぐために`DATA_DIR`環境変数を設定してください。各プラットフォーム向けの例を示します：

- **macOS/Linux**:  
  ```bash
  DATA_DIR=~/.sage-open-webui uvx --python 3.11 sage-open-webui@latest serve
  ```

- **Windows**:  
  ```powershell
  $env:DATA_DIR="C:\sage-open-webui\data"; uvx --python 3.11 sage-open-webui@latest serve
  ```

### `pip`を使用したインストール

Pythonのパッケージマネージャー`pip`でSage WebUIをインストールするユーザーには、**`uv`や`conda`などのPythonランタイムマネージャーの使用を強く推奨します**。これらのツールはPython環境を効果的に管理し、競合を回避するのに役立ちます。

開発環境はPython 3.11です。Python 3.12でも動作するようですが、十分にテストされていません。Python 3.13は完全に未テストです—**自己責任で使用してください**。

1. **Sage WebUIのインストール**:  

   ターミナルを開き、以下のコマンドを実行します：  
   ```bash
   pip install sage-open-webui
   ```

2. **Sage WebUIの起動**:  

   インストール後、以下のコマンドでサーバーを起動します：  
   ```bash
   sage-open-webui serve
   ```

### Sage WebUIの更新

最新バージョンに更新するには、以下のコマンドを実行します：

```bash
pip install --upgrade sage-open-webui
```

この方法では、必要な依存関係をすべてインストールし、Sage WebUIを起動するため、シンプルで効率的なセットアップが可能です。インストール後、Sage WebUIには[http://localhost:8080](http://localhost:8080)でアクセスできます。お楽しみください！😄
