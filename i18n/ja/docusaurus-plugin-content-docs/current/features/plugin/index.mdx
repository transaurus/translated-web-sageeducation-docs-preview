# 🛠️ ツールと機能

Sage WebUIを初めて使う方、あるいは既に利用している方でも、「ツール」「機能」「パイプライン」といった用語に戸惑っているかもしれません。これらは一見すると難解な技術用語のように聞こえますが、心配はいりません。ここでは一つひとつを分かりやすく、段階的に解説します。読み終わる頃には、これらの用語の意味や仕組み、そしてそれらが思ったほど複雑ではないことが理解できるでしょう。

## 概要

- **ツール**：AIの能力を拡張し、天気や株価など現実世界のリアルタイムデータを収集できるようにします。
- **機能**：Sage WebUI自体の能力を拡張し、カスタムAIモデルの追加やユーザビリティの向上（カスタムボタンやフィルターの作成など）を可能にします。
- **パイプライン**：高度なユーザー向けで、Sage WebUIの機能をAPI互換のワークフローに変換するためのものです。主に重い処理を別のマシンにオフロードする場合に使用します。

ツールと機能の利用は簡単です。コアシステムに既に組み込まれており、**ボタンをクリック**して**コミュニティから直接インポート**するだけです。コーディングや深い技術的知識は不要です。

## 「ツール」と「機能」とは？

まず、**Sage WebUI**を「ベース」ソフトウェアと考えてみましょう。これは大規模言語モデル（LLM）を使用するための多くのタスクを実行できます。しかし、時には**標準では提供されていない**追加機能や能力が必要になることがあります。ここで**ツール**と**機能**が活躍します。

### ツール

**ツール**は、LLMがテキスト処理以上のことを可能にするエキサイティングな機能です。LLM単体では持たない**外部能力**を提供します。

#### ツールの例：

LLMとチャットしている際、最新の天気予報や株価をリアルタイムで知りたいとします。通常、LLMは事前に学習した知識しか使えないため、これらを取得できません。ここで**ツール**が役立ちます！

- **ツールはプラグインのようなもの**で、LLMが**現実世界のリアルタイムデータ**を収集するために使用します。例えば「天気ツール」を有効にすると、モデルはインターネットからライブの天気データを取得し、会話中に表示できます。

ツールは本質的に、AIに外部世界とやり取りするための**能力**を与えるものです。これらを追加することで、LLMは会話の文脈に基づいて有用な情報を「取得」したり、専門的なタスクを実行したりできます。

#### ツールの例（LLMの能力拡張）：

1. **リアルタイム天気予報** 🛰️
2. **株価取得** 📈
3. **フライト追跡情報** ✈️

### 機能

**ツール**が会話中にAIによって使用されるのに対し、**機能**はSage WebUI自体の能力を拡張またはカスタマイズします。ツールが追加の材料だとすると、機能はそれらを組み合わせたり、形作ったり、味わったりする調理技術のようなものです。料理が手順の習得に関わるように、機能はSage WebUIに対して同じことを行います。

#### 詳しく説明すると：

Sage WebUI内に新機能を追加します。例えば、改良されたAIモデル、カスタムボタン、強化されたフィルタリング機能などです。
プラットフォームのインターフェース、動作、ロジックを調整・カスタマイズし、ニーズに合わせて最適化します。

具体的な例を挙げます：

- 頻繁に使うコマンド用のカスタムツールバーボタンを作成する。
- スパムメッセージをブロックする効果的なフィルターを開発する。

これらの能力は機能によって提供され、Sage WebUIを自分の要件に合わせて形作ることができます。

### 違いのまとめ：

- **ツール** は、AIがデフォルトの能力を超えて**より多くのことを行える**ようにするものです（例えば天気をチェックしたり、外部データを使ったカスタムタスクを実行したり）。
- **関数** は、WebUI自体が**より多くのことを行える**ようにするもので、新しいAIモデルの追加やデータをスマートにフィルタリングする方法の作成などが可能です。

どちらも**プラグ可能**に設計されており、コミュニティからワンクリックで簡単にシステムにインポートできます！また、他のSageユーザーと共有することも可能です。

🎉 新しいツールや関数を作成して楽しむ場合を除き、何時間もコーディングや調整に時間を費やす必要はありません。

## パイプラインとは？

そして、**パイプライン**があります…ここから少し技術的な話になりますが、心配はいりません。

**パイプライン**は、Sage WebUIのすべての機能を**OpenAIのAPIシステムと相互運用可能**にするための取り組みの一部です。基本的に、**ツール**と**関数**が既に行えることをさらに柔軟に拡張し、機能をOpenAI API互換の形式に変換できます。🧠

### ただし、重要な点は…

非常に高度なセットアップを扱う場合を除き、**パイプラインは必要ない**でしょう。

- **パイプラインは誰向け？** 一般的に、**エキスパート**やより複雑なユースケースを実行する人向けです。
- **いつ必要？** プライマリSage WebUIインスタンスから別のマシンに処理をオフロードしたい場合（プライマリシステムに過負荷をかけないため）。

初心者や中級ユーザーの場合、ほとんどのケースでパイプラインを気にする必要はありません。**ツール**と**関数**がSage WebUIにもたらす利点を楽しむことに集中しましょう！

## 試してみたいですか？🚀

Sage WebUIに飛び込んで、コミュニティセクションに移動し、**天気更新**のようなツールをインポートしたり、関数を使ってツールバーに新しい機能を追加してみてください。これらのツールを探索することで、Sage WebUIがいかに強力で柔軟かを実感できるでしょう！

🌟 学ぶことは常にありますので、好奇心を持ち続け、実験を続けましょう！
