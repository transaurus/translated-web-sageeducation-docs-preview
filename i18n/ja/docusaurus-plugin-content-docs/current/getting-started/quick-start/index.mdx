:::info[**ユーザーロールとプライバシーに関する重要事項:**]

- **管理者アカウントの作成:** Sage WebUIで最初に作成されたアカウントは**管理者権限**を取得し、ユーザー管理とシステム設定を制御します。
- **ユーザー登録:** その後サインアップしたアカウントは**保留中**ステータスで始まり、アクセスには管理者の承認が必要です。
- **プライバシーとデータセキュリティ:** **すべてのデータ**（ログイン情報を含む）はデバイス上に**ローカルで保存**されます。Sage WebUIは**厳格な機密保持**と**外部へのリクエストなし**を保証し、プライバシーとセキュリティを強化します。
  - **すべてのモデルはデフォルトで非公開です。** モデルはグループ経由で共有するか、公開設定にする必要があります。モデルがグループに割り当てられた場合、そのグループのメンバーのみが閲覧可能です。モデルが公開設定の場合、インスタンス上の誰もが閲覧できます。

:::

以下のインストール方法からお選びください:

- **Docker:** **公式サポートされており、ほとんどのユーザーに推奨**
- **Python:** リソースが限られた環境や手動セットアップを希望する場合に適しています
- **Kubernetes:** スケーリングとオーケストレーションが必要なエンタープライズ環境に最適

## 次のステップ

インストール後は以下にアクセスしてください:

- [http://localhost:3000](http://localhost:3000) Sage WebUIにアクセスします。
- またはPythonデプロイメントの場合は [http://localhost:8080/](http://localhost:8080/) にアクセスします。

これでSage WebUIの使用を開始できます！

## Sage WebUIとOllamaの連携

Sage WebUIをOllamaと連携させる場合は、[Ollama開始ガイド](/getting-started/quick-start/starting-with-ollama)を参照して、Sage WebUIでOllamaインスタンスを管理する方法を学んでください。
